hexo博客使用指南与问题

Hexo博客的本地搭建

hexo博客的本地搭建基本上通过官方文档就可以完成了。
此外也参考了一些博客Hexo系列 | Hexo的基本使用和hexo史上最全搭建教程
本地基本上只要hexo g和hexo server确认是否能渲染即可
可能需要注意一下categories和tags的区别（请看官方文档）

Hexo博客&github pages

理论上只要跟着官方文档的“一键部署”步骤就行了（
但是这次有个比较特别的情况，我之前有一个github账号，这次为了建立这个博客重新创建了一个github账号，但ssh -T git@github.com的结果是原先的账号。
清除了一些git的缓存并且做出了一些尝试（真的有用吗？），发现有用但用处不大（

  1. git config core.sshCommand "ssh -i ~/.ssh/CELFblog_rsa"我希望它能在当前目录下用CELFblog的rsa，但是不知道为啥没有用
  2. 改变~/.ssh/config的值为

    Host github.com
    HostName github.com
    IdentityFile ~/.ssh/CELFblog_rsa

    这个真有用，ssh -T git@github.com的结果变为新账号
  3. 在ssh结果默认新账号之后，试图用hexo d进行部署，发现每次都需要输入账号名和密码（不能用注册的密码，需要使用github的Github Access Token(GAT)登录，太烦了。各种git config --unset --global啥的，credential.helper也试了但是没有用
  4. 最后的解决方法是将_config.yml改为 虽然能用但心里不太舒服，毕竟是明文存储在本地，而且还是不会切换账号（

还有一个注意事项是在repository的settings的github pages中，deploy from a branch记得选（要么是master要么是gh-pages）

To train a model using NLTK for sorting reviews into three classes (-1, 0, 1), you can follow these general steps:

Data Preparation:

Prepare your review data, ensuring it is labeled with one of the three classes (-1, 0, 1).
Split your data into a training set and a test set.

Text Preprocessing:

Tokenize your text data (split it into individual words or tokens).
Optionally, remove stopwords (common words that may not carry much meaning) and perform stemming or lemmatization to reduce words to their base form.

Feature Extraction:

Convert your text data into a numerical representation that can be used by machine learning algorithms. One common approach is using TF-IDF (Term Frequency-Inverse Document Frequency) to convert text into numerical features.

Model Training:

Choose a machine learning model suitable for text classification, such as Naive Bayes, SVM, or logistic regression.
Train your model using the training set.

Model Evaluation:

Evaluate the performance of your model using the test set.
You can use metrics such as accuracy, precision, recall, or F1-score to evaluate the performance of your model.
